# /wiki maintain — Vault Maintenance Skill

> Trigger: user types `/wiki maintain` in a Claude Code session.
> This skill is automatically loaded alongside `SKILL.md`.

---

## What This Skill Does

Performs a semantic-level audit of the wiki vault. It finds:

1. **Contradictions** — notes that make conflicting claims about the same concept
2. **Stale TODOs** — notes that reference a TODO item that has since been completed
3. **Orphan concepts** — terms mentioned 5+ times across notes but having no defining node
4. **Broken cross-refs** — `[[wikilinks]]` pointing to renamed or deleted nodes
5. **Missing definitions** — doc references to entities not yet in the graph

Output: `wiki-out/maintain-report-<YYYYMMDD-HHMM>.md`

---

## Inputs

| Input | Default | Description |
|-------|---------|-------------|
| Vault path | `./wiki-out/vault` | Directory containing `*.md` vault notes |
| `--since <date>` | *(none)* | Limit stale TODO scan to notes modified after this date (ISO format: `YYYY-MM-DD`) |

Example invocations:
```
/wiki maintain
/wiki maintain --since 2025-01-01
/wiki maintain /path/to/custom/vault
```

---

## Anti-Hallucination Rule

**CRITICAL: Every cited node, label, file name, or page number in the report MUST come from CLI tool output or direct file reads — never from memory or inference.**

Before writing any finding:
1. Run the relevant `llm-wiki query` command
2. Read the actual note file if citing content
3. Only cite what is in those outputs

If unsure whether a node exists: run `llm-wiki query node <label>` and check the output.

---

## Step-by-Step Procedure

### Step 1 — Read the vault index

```bash
cat wiki-out/vault/index.md
```

Record the list of categories and note counts. This is your map of what exists.

### Step 2 — Sample notes per category

For each category in the index, read up to 10 notes:

```bash
ls wiki-out/vault/<category>/
# then read representative files
```

Note any:
- Contradictory claims (same concept described differently in 2+ notes)
- TODO/FIXME markers that reference code or decisions that appear resolved
- References to concepts that seem important but lack a dedicated node

### Step 3 — Run orphan check

```bash
llm-wiki query orphans
```

Record ALL node labels in the output. These are concepts with no connections — candidates for:
- Linking to existing nodes via `llm-wiki note --link`
- Merging if duplicate of another node
- Deletion if truly irrelevant

Also run with image hubs included for completeness:

```bash
llm-wiki query orphans --include-hubs
```

### Step 4 — Run stale reference check

```bash
llm-wiki query stale-refs wiki-out/vault
```

Each line `<file>:<line> -> [[Missing]]` is a broken wikilink. These arise when:
- A node was renamed and old links weren't updated
- A note was deleted but cross-references remain
- A concept was referenced before it was added to the graph

### Step 5 — Cross-check with existing query commands

For any node or concept flagged in steps 3–4, verify with:

```bash
llm-wiki query node <label>         # confirm existence and details
llm-wiki query neighbors <label>    # check actual connections
llm-wiki query search <term>        # find related nodes by keyword
llm-wiki query path <A> <B>         # check if a path exists when expected
llm-wiki query stats                # overall health numbers
```

### Step 6 — Estimate staleness with git blame

For notes referencing source files, estimate age:

```bash
git blame <source_file> | head -20
git log --since="90 days ago" --oneline <source_file>
```

If a source file has had no commits in 90+ days, any TODO referencing it is likely stale.

### Step 7 — Write the report

Create the report file:

```bash
TIMESTAMP=$(date +%Y%m%d-%H%M)
REPORT="wiki-out/maintain-report-${TIMESTAMP}.md"
```

Use the **Output Template** below. Fill every section — write "None found" for empty categories.

## Severity Levels

| Level | Meaning | Action |
|-------|---------|--------|
| `error` | Broken ref or direct contradiction — graph inconsistent | Fix before next rebuild |
| `warn` | Potential issue, requires human judgment | Review and triage |
| `info` | Observation only — low confidence or minor | Optional |

## Examples

**Contradiction:** `vault/core/Cache.md:4` says "invalidated by SHA256 mismatch"; `vault/ops/Deploy.md:11` says "invalidated by mtime only" → severity `warn`, reconcile via `llm-wiki note`.

**Broken cross-ref:** `llm-wiki query stale-refs wiki-out/vault` reports `api/Auth.md:7 -> [[OldAuthModule]]` — node was renamed to `AuthMiddleware`. Fix: update the wikilink.

**Orphan concept:** `llm-wiki query orphans` reports `BatchProcessor` (degree 0). Appears in 7 notes but no defining node. Run `llm-wiki query search BatchProcessor`, then `llm-wiki note` to create one.

**Page citation:** `llm-wiki query node ArchitectureDecisions` shows `page: 12` (PDF source). Cite as: "ArchitectureDecisions (docs/arch.pdf, page 12) references EventBus — no EventBus node found." Only cite `page:` when the field is present in query output.

---

## Limitations

- **Age estimation:** Use `git blame` on source files as staleness proxy. Non-git dirs: mark all TODOs as `info`.
- **Contradiction detection is semantic:** False positives possible — always read both notes before reporting.
- **Orphan ≠ useless:** Freshly-added nodes are briefly orphan. Flag only nodes orphan for 2+ rebuilds.
- **Read-only:** Never mutates vault or graph. Fix via `llm-wiki note` / `llm-wiki add`, then `llm-wiki .`.

## Quick Reference: Commands Used in This Skill

```bash
llm-wiki query orphans                  # isolated nodes (default: exclude image hubs)
llm-wiki query orphans --include-hubs   # include image hub nodes
llm-wiki query stale-refs <vault>       # broken wikilinks
llm-wiki query node <label>             # node details (incl. page citation if PDF)
llm-wiki query neighbors <label>        # direct connections
llm-wiki query search <term>            # keyword search
llm-wiki query path <A> <B>             # shortest path
llm-wiki query stats                    # graph health numbers
llm-wiki query gods                     # most-connected nodes
llm-wiki lint                           # overall graph health check
git blame <file>                        # estimate source file age
git log --since="90 days ago" <file>    # recent activity on source
```
